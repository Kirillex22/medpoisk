Прототип системы для интеллектуального поиска медицинских статей в PubMed с использованием GigaChat для генерации поисковых запросов и ранжирования результатов.

## ✨ Возможности

- Принимает запрос на естественном языке (русском или английском)
- Генерирует 3 варианта структурированных поисковых запросов для PubMed через GigaChat
- Выполняет поиск по всем вариантам, объединяет результаты
- Получает полные метаданные статей (название, авторы, журнал, дата, DOI, страницы, том/выпуск)
- Загружает аннотации (abstract) для каждой статьи через EFetch
- **Ранжирует результаты по релевантности исходному запросу с помощью GigaChat**
- Кэширует результаты на клиенте
- Позволяет оценивать статьи (like/dislike) для обратной связи

## 📦 Установка и запуск

### 1. Скачайте репозиторий и перейдите в папку с ним.

### 2. Настройка переменных окружения

Отредактируйте файл `.env.example` в корне проекта, заполнив своими данными:

```env
# GigaChat
GIGACHAT_CREDENTIALS=your-credentials-here
GIGACHAT_MODEL=GigaChat
GIGACHAT_TEMPERATURE=0.3
GIGACHAT_MAX_TOKENS=1024
```

После этого переименуйте его в `.env`

**Где взять credentials:**
- `GIGACHAT_CREDENTIALS` — получить в личном кабинете Сбера (авторизационные данные для GigaChat)

### 3. Запуск 

#### Для Windows
Просто выполните скрипт `start.bat` двойным щелчком. Он создаст виртуальное окружение, установит зависимости и запустит сервер.

#### Для Linux / MacOS (или вручную)
```bash
python -m venv .venv
source .venv/bin/activate   # Linux/Mac
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
uvicorn src.main:app --reload
```

После запуска страница с чатом будет доступна по адресу `http://127.0.0.1:8000`.  
Документация API (Swagger): `http://127.0.0.1:8000/docs`

## 🧠 Как это работает

1. Пользователь вводит запрос, например: *"лечение резистентной гипертонии у пациентов с диабетом"*
2. Бэкенд отправляет этот запрос в GigaChat вместе с системным промптом из файла `prompts/query_generate.prompt`, требующим сгенерировать 3 варианта структурированных запросов для PubMed (с полями MeSH, Publication Date и т.д.)
3. Полученные запросы отображаются как кликабельные карточки
4. При клике на запрос бэкенд выполняет поиск в PubMed:
   - `esearch.fcgi` — получает список PMID
   - `esummary.fcgi` — получает метаданные (название, авторы, журнал и т.д.)
   - `efetch.fcgi` с `rettype=abstract` — загружает аннотации в XML
5. **Ранжирование**: полученные статьи (вместе с исходным запросом) отправляются в GigaChat, который возвращает порядок PMID от наиболее релевантного к наименее
6. Отсортированные статьи отображаются в чате, при наведении показывается аннотация
7. Пользователь может поставить лайк/дизлайк (оценка пока только логируется)

## ⚙️ Настройка под себя

- **Температура GigaChat** — измените `GIGACHAT_TEMPERATURE` в `.env` (0.0 — строго, 1.0 — креативно)
- **Поведение генерации запросов** — отредактируйте файл `prompts/query_generate.prompt`. Вы можете изменить инструкции, примеры или требования к формату, не касаясь кода.
- **Поля статей** — можно расширить в модели `Article` и парсинге `esummary` в `src/main.py`

## 📌 Примечания

- GigaChat требует авторизацию через credentials (не токен). Убедитесь, что в `.env` указан корректный `GIGACHAT_CREDENTIALS`.
- Аннотации могут отсутствовать для некоторых статей (тогда показывается "Аннотация отсутствует").
- Для больших нагрузок на NCBI рекомендуется получить `api_key` и добавить его в код (см. документацию Entrez).
- Проект находится в стадии прототипа, возможны изменения.
